# infrastructure/

Terraform for the Aegis Voice Guide AWS deployment.
See repo-root `agent/AGENT_V2.md` §8 for the full architectural context.

## What this builds

| Resource | Purpose |
|---|---|
| `aws_security_group` | SSH 22 from `var.admin_ip_cidr` only; all egress |
| `tls_private_key` + `aws_key_pair` + `local_sensitive_file` | RSA-4096 EC2 keypair, PEM written 0400 to `aegis-voice-guide.pem` (gitignored) |
| `aws_iam_role` + `aws_iam_role_policy` + `aws_iam_instance_profile` | Least-privilege role: Secrets Manager read on `aegis-voice-guide/*`, CloudWatch Logs put |
| `aws_secretsmanager_secret(_version)` × 6 (+ optional Gemini) | Runtime provider keys pushed from `agent/.env.local` |
| `aws_instance` t3.medium + `aws_eip` | The agent host, with stable public IP |
| Encrypted gp3 root volume, IMDSv2-only | Prevents SSRF cred theft + encrypted at rest |
| `aws_cloudwatch_metric_alarm` (billing) | Fires at $200 (warn) via SNS topic |
| `aws_lambda_function` autostop + `aws_cloudwatch_event_rule` | Stops the instance if CPU < 5% for 30 min |
| `aws_cloudwatch_log_group` | Agent logs + Lambda logs |

## Operations

All commands are run from the repo root unless stated otherwise.

```bash
# 1. First-time setup — non-billable. Reviews what will be created.
./infrastructure/scripts/plan.sh

# 2. After explicit cost approval — provisions everything. BILLABLE.
./infrastructure/scripts/apply.sh

# 3. Push code + run ingest + start the agent service on the box.
./infrastructure/scripts/deploy.sh

# 4. Day-to-day:
./infrastructure/scripts/start.sh    # wake the instance before an interview
./infrastructure/scripts/ssh.sh      # interactive SSH session
./infrastructure/scripts/status.sh   # state + recent agent logs
./infrastructure/scripts/stop.sh     # save money after the demo
```

## Where secrets live

| What | Where | Permissions | Gitignored |
|---|---|---|---|
| AWS provisioning keys (this account) | `infrastructure/.env.aws.local` | `0600` | ✅ |
| Agent runtime keys (Deepgram, Cartesia, Groq, LiveKit, optional Gemini) | `agent/.env.local` locally; `aws_secretsmanager_secret` on EC2 | `0600` / IAM-scoped | ✅ |
| EC2 SSH private key | `infrastructure/aegis-voice-guide.pem` (generated by `terraform apply`) | `0400` | ✅ |
| Terraform state | `infrastructure/terraform.tfstate` | local backend for now | ✅ |

> ⚠️ The AWS keys were pasted in chat during build setup. **Rotate them** in AWS Console → IAM → Users → Security credentials after this session.

## Cost shape

| Item | ~ $/mo ⚠️ |
|---|---|
| `t3.medium` always-on | ~$30 |
| `t3.medium` stopped overnight + weekends (~10 h/day × 22 days) | ~$10 |
| gp3 EBS 30 GB (charged even when stopped) | ~$3 |
| Elastic IP (attached to running instance) | $0 |
| Secrets Manager (6 secrets × $0.40) | ~$2.4 |
| CloudWatch logs (very small) | ~$1 |
| Lambda autostop (1 invoke / 5 min × 30 days) | <$0.10 |
| Data transfer (demo traffic) | ~$1–2 |
| **Total stop-overnight** | **~$15–20** |
| **Total always-on** | **~$35–40** |

Hard budget cap: **$250/mo** — enforced by the billing alarm.
Realistic target for a portfolio: **$15–20/mo** with stop-overnight discipline + auto-stop on idle.

## Safety / linting

```bash
cd infrastructure
terraform fmt -recursive
tflint --recursive
tfsec .
```

Run all three before any `apply`. Any tfsec finding that's intentionally accepted is documented with a `# tfsec:ignore:<rule>` inline comment + reason.
